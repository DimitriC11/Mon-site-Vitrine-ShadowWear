═══════════════════════════════════════════════════════════════════════════════
💾 STOCKAGE DES DONNÉES
═══════════════════════════════════════════════════════════════════════════════

localStorage (PERSISTANT entre les sessions)
  • Clé : "users"
  • Contenu : Tableau JSON de tous les utilisateurs
  • Survit à la fermeture du navigateur
  • Stocké localement, pas d'envoi au serveur

sessionStorage (TEMPORAIRE - session seule)
  • Clé : "currentUser"
  • Contenu : Profil de l'utilisateur connecté
  • Supprimé à la fermeture de l'onglet
  • Utilisé pour vérifier la connexion

Structure d'un utilisateur :
{
  "id": 1708617600000,          // Timestamp unique
  "name": "Jean Dupont",         // Nom complet
  "email": "jean@example.com",   // Email unique
  "password": "password123",     // Mot de passe (en clair)
  "role": "user",                // "user" ou "admin"
  "createdAt": "2025-02-22T..."  // ISO date
}

🔧 FONCTIONS UTILES (js/auth.js)
═══════════════════════════════════════════════════════════════════════════════

Enregistrement :
  registerUser(name, email, password, role='user')

Authentification :

Admin :
  createAdminAccount(name, email, password)
  promoteToAdmin(email)
  demoteFromAdmin(email)

Gestion :

  changePassword(email, newPassword)

Debug :

  debugClearAllUsers()         - Supprimer TOUS les utilisateurs

═══════════════════════════════════════════════════════════════════════════════
⚠️ LIMITATIONS (à noter)
═══════════════════════════════════════════════════════════════════════════════

1. SÉCURITÉ
   ⚠️ Les mots de passe sont stockés en CLAIR dans localStorage
   → En production : utiliser bcryptjs pour les hasher

2. PERSISTANCE
   ⚠️ Les données sont perdues si on vide le cache du navigateur
   → En production : synchroniser avec une base de données

3. VALIDATION
   ⚠️ La validation est faite côté client (peut être contournée)
   → En production : ajouter une API backend pour la validation

4. AUTHENTIFICATION
   ⚠️ Pas de token JWT ou de session serveur
   → En production : implémenter un système Backend d'authentification

❓ QUESTIONS FRÉQUENTES
═══════════════════════════════════════════════════════════════════════════════

Q : Où sont stockées les données ?
R : Dans localStorage du navigateur. Accessible via DevTools > Application > localStorage

Q : Les données sont-elles transférées au serveur ?
R : Non, elles restent uniquement dans le navigateur de l'utilisateur.

Q : Puis-je exporter les données ?
R : Oui ! Panel admin > "Exporter les données" → fichier JSON téléchargé.

Q : Comment supprimer toutes les données ?
R : Panel admin > "Supprimer TOUT" ou console > localStorage.clear()

Q : Les utilisateurs peuvent-ils être promis admin ?
R : Oui ! Panel admin > tableau utilisateurs > modifier rôle.

Q : Comment ça marche hors ligne ?
R : Parfaitement ! Les pages HTML statiques et localStorage travaillent hors ligne.
